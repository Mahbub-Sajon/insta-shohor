Question: 2. What does javascript event loop do?
Answer: As we know javascript is a single threded language, which means it can execute an event at a time in the mean time other events stay in queue list which is known as event queue. And if the event is a function and that function calls another function then it waits untill the first function is done which is basically a call stack and this whole process can be determined as a synchronous event. Well, there is another thing which is asynchronous event that can execute multiple events at a time. Well, not exactly like this cause our javascript is still a single threaded language so what happens is, it starts executing a function and while executing, it can keep that in a queue through call back and can go to the next event. After finishing the first function's execution it can show the result while the second one is already processing or maybe it is also done within the time. So, asynchronous is basically less time consuming even if it is single threaded. Question: 3. Differences between local storage and session storage.
Answer: Local storage- #The data saved in a local storage can not be deleted automatically or by itself. It doesn't work for any specific tab of the browser but for any tabs we use. Local storage data can sill be there even if we turn off the device. But if we use incognito mode of a browser the data get cleared when we close the tab. The storage capacity is 10 mb. Session Storage- # In session storage the data get deleted automatically when we close the tab or the browser or turn off the device. Session storage only works on specific tab. New session storage is created each time we open a new tab. The storage capacity is 5 mb.
